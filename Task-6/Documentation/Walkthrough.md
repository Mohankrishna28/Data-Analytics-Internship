## 2. Running the Project

### Phase 1: Prototype Verification
You can open and execute all cells in the Jupyter notebook `Analysis.ipynb` to verify the cleaning logic. This will print the filtered category shape and export `Graph1.png` to the screenshots folder.

### Phase 2: Launching the Dashboard
To run the Streamlit server locally:
```bash
streamlit run Task-6/dashboard.py
```
By default, the server will open at `http://localhost:8501`.

## 3. Visual Walkthrough & Validation

### 🔒 Locked State (Outside 1 PM – 2 PM IST)
If you open the dashboard outside the active hours (1:00 PM – 2:00 PM IST), the visualization is hidden. A red locked notice is displayed with a digital clock and a real-time countdown timer indicating when the dashboard will next unlock.

![Locked View Notice](c:/Users/mohan/Downloads/Play%20Store%20Reviews/Task-6/Screenshots/Locked_View.png)

---

### 🔓 Unlocked State (Bypassed via Simulator / Between 1 PM – 2 PM IST)
By toggling **"Time Simulation Mode"** in the sidebar to **"Force Unlocked (Simulate 1:30 PM IST)"**, the dashboard unlocks.
It presents sleek KPI metric cards, an interactive sidebar with sliders for all thresholds, the dual-axis chart comparing Free/Paid installs and revenue, and a searchable preview of the underlying dataset.

![Unlocked Dashboard View](c:/Users/mohan/Downloads/Play%20Store%20Reviews/Task-6/Screenshots/Dashboard_View.png)
